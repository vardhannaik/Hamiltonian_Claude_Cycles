Cooperative Cycle Coverage
on Toroidal Grids

Diagonal Leaders, Golden Scouts, and Late-Stealing in n Dimensions

Subject · Multi-Agent Coverage Dimensions · 3D, 4D Grid size · m = 5 Status · Preprint
Abstract We present a multi-agent cooperative coverage algorithm for n-dimensional toroidal grids where each agent follows a pre-planned path but dynamically steals vertices from its leaders' future schedules. A diagonal Hamiltonian (C0) sweeps the grid in uniform fiber planes; golden-ratio scouts (C1, C2, …) first fill pure gaps — vertices no leader will ever visit — then prioritise stealing the latest-scheduled vertices from their leaders, since those are visited last and cause the most delay. Applied to m=5 grids, the algorithm achieves 50.4% efficiency in 3D (63/125 steps) and 56.6% in 4D (354/625 steps), compared to 73.6% and 69.1% for pure-existence scoring.

1 · Background

Consider an m×m×···×m toroidal grid: each coordinate wraps modulo m, and two vertices are adjacent if their taxicab distance is exactly one. Multiple agents must collectively visit every vertex at least once, minimising the cover step — the first timestep at which the union of all visited vertices equals the full grid.

Each agent moves one step per timestep. The challenge is coordination: without it, agents cluster and revisit the same vertices. The key insight is role asymmetry.

C0 is a strict leader following a pre-planned diagonal path. Scouts observe C0's future schedule and adaptively choose vertices that no leader will reach — or will reach only very late.

This allows scouts to make informed decisions with zero runtime communication — they need only read-access to the leaders' pre-planned sequences.

2 · Path Construction

Diagonal Hamiltonian (C0)

The diagonal path sweeps fiber planes s = (i+j+k+···) mod m in strict order. Within each fiber, a Warnsdorff tiebreaker prefers vertices with fewer unvisited neighbours, preventing dead ends.

Key structural property: C0 visits vertices with coordinate sum s=0 first, s=1 next, and so on. The tail of C0's schedule contains vertices with the largest coordinate values — concentrated in the high-index corner (m−1, m−1, …) of the grid. This is measurable: in 4D at m=5, late vertices have average coordinate sum 10.0 versus 6.0 for early vertices — a gap of 4.0. In 3D the gap is smaller (7.08 vs 5.74 = 1.3), which is why the optimal late-steal window is narrower.

def getD(i, j, k, m):
    """Direction selector based on fiber s = (i+j+k) % m."""
    s = (i + j + k) % m
    if s == 0:
        return "012" if j == m - 1 else "210"
    elif s == m - 1:
        return "120" if i > 0 else "210"
    else:
        return "201" if i == m - 1 else "102"

def diagonal3d(m):
    i, j, k = 0, 0, 0
    path = [(0, 0, 0)]

        i, j, k = nb[int(d[0])]
        path.append((i, j, k))
    return path

Golden-Ratio Path (Scouts)

Each scout follows a path constructed by recursively splitting the grid along its longest axis at the golden ratio φ = (1+√5)/2, then stitching the two halves together at an adjacent boundary. The result is a Hamiltonian path with locally clustered, globally spread structure — exactly what is needed to fill the gaps left by the diagonal leader.

φ = (1 + √5) / 2 ≈ 1.618  ·  split_index = ⌊dim / φ⌋

3 · Role-Aware Simulation

V3 · Pure Existence Scoring

The first working version (V3) assigns each scout a tier based purely on whether any leader will ever visit a candidate vertex:

C1 → tier 2: C0 will never visit v ← pure gap
tier 1: C0 will visit v
C2 → tier 3: neither C0 nor C1 visits ← pure gap
tier 2: C0 skips, C1 will come
tier 1: C0 will visit

V3 achieves 92/125 steps in 3D and 432/625 in 4D — already a significant improvement over naive approaches, because scouts fill the structural gaps left by C0's fiber-ordered sweep.

V6 · Existence Tiers + Late-Stealing

V6 keeps V3's existence tiers as the primary signal and adds a late-steal sub-tier: vertices in the last LATE_FRAC of a leader's remaining schedule receive higher priority than vertices the leader visits soon. The threshold is dynamic — recomputed each step as a fraction of C0's remaining schedule, not its total.

C1 → tier 3: C0 will never visit ← pure gap (unchanged from V3)
tier 2: C0 visits, but LATE (last ~10–19%) ← steal the tail
tier 1: C0 visits soon ← let C0 handle it
C2 → tier 4: pure gap — neither C0 nor C1 ← highest priority
tier 3: C0 skips, C1 visits LATE ← steal C1's tail
tier 2: C0 skips C1-soon, or C0-late
tier 1: C0 visits soon

LATE_FRAC = 0.095   # 3D tuned  (use 0.187 for 4D)

for t in range(total):
    c0_step = t + 1
    c0_rem  = total - c0_step
    thr     = int(c0_rem * (1 - LATE_FRAC))   # dynamic threshold

    def lateness(smap, v, cur):
        vi = smap.get(v, -1)

The optimal late-steal window scales with the tail concentration of the diagonal path — how much larger the average coordinate sum is for late vertices versus early ones. 5 · Why Late-Stealing Works

The diagonal C0 path starts at (0,0,0,…) and processes vertices in order of increasing coordinate sum modulo m. Due to the toroidal structure, vertices near (m−1, m−1, m−1, …) — the high-index corner — accumulate in C0's tail schedule. They are visited last because the Warnsdorff tiebreaker systematically defers them: other vertices have more unvisited neighbours and are therefore preferred.

Stealing these tail vertices achieves two things simultaneously:

1. Direct time compression. A vertex C0 would have visited at step t is instead visited at step t' << t. The cover time drops by (t − t') if this was the last uncovered vertex.

2. Cascade compression. When scouts consume C0's late vertices, C0 arrives at an already-covered position and must proceed — effectively shortening C0's useful work and making its remaining schedule denser with truly uncovered vertices. The coverage curve accelerates.

The scouts are not just filling gaps — they are reshaping the effective coverage frontier so that C0's structured sweep finishes sooner.

6 · Extension to nD

The framework extends naturally to n dimensions. Each new dimension requires one additional scout (Cn−1) with awareness of all previous leaders. The tier count grows by one per scout, and the late-steal threshold likely requires re-tuning as the tail concentration of the diagonal path changes with dimension.

In n dimensions the scoring for scout Ck is:

n+1 → none of C0…C(k−1) will ever visit v ← pure gap
n → C0…C(k−2) skip, C(k−1) visits LATE ← steal C(k−1)'s tail
n−1 → C0…C(k−3) skip, C(k−2) visits LATE
… … (one tier per leader, alternating gap / late-steal) …
1 → C0 visits soon ← lowest priority
